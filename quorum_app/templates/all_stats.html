Questions & Answers

1. Discuss your strategy and decisions implementing the application. Please, consider time complexity, effort cost, technologies used, and any other variable that you understand important in your development process.

Answer: My strategy involved creating Django models to represent the data, namely Legislator, Bill, Vote, and VoteResult. I used Django's ORM to perform efficient queries, reducing time complexity. The effort cost was minimized by leveraging the existing Django framework and its features. Technologies used include Django, Python, and HTML for web development. The decision to structure the application with separate models for each data entity allowed for better organization and readability of the code.

2. How would you change your solution to account for future columns that might be requested, such as “Bill Voted On Date” or “Co-Sponsors”?

Answer: To accommodate future columns, I would extend the existing models by adding fields for the new data, such as "voted_on_date" in the Bill model or creating a new model for Co-Sponsors. I would then update the data import process and adjust the queries in views accordingly. This approach ensures flexibility and scalability for handling additional data columns.

3. How would you change your solution if instead of receiving CSVs of data, you were given a list of legislators or bills that you should generate a CSV for?

Answer: If provided with a list of legislators or bills to generate a CSV, I would create a new Django management command to export the required data to CSV format. This command would take input parameters, such as a list of legislator IDs or bill IDs, and dynamically generate the CSV file. The Django CSV module or the CSV writer in Python could be used for this purpose.

4. How long did you spend working on the assignment?

Answer: I spent approximately 2 hours working on the assignment, including data modeling, implementation, testing, and documentation.
